### Подготовка
Я выполняю это на ванильной TOS 6.0, но думаю на условной DSM, будет все плюс минус так же, даже проще. 
1. Проверяем установлен ли docker. Если нет, его легко натыкать в GUI
```bash
docker --version
```
2. Если не используете Wireguard пропускайте этот шаг, в обратном случае добро пожаловать в TOS. Поддержка WG у ядра есть, [но она выключена](https://tmnascommunity.eu/download/wireguard/), проверяем, так ли это. В случае если модуль не загружен, включим. После ребутните NAS.
```bash
lsmod | grep wireguard
#если в воде пусто, то включим модуль WG
echo wireguard >> /etc/modules-load.d/wireguard.conf
```
3. Любым удобным способо создайте пользователя либо через web-интерфейс, либо через  терминал. Для данной статьи я создал пользоваетля `media`. Узнайте uid и gid данного пользователя 
```bash
id media
uid=1004(media) gid=1004(media) groups=1004(media),997(allusers),10002(qbittorrent)
```

В условиях ограниченности ресурсов, рекомендую использовать Qbittorent в docker, т.к. ограничить ресуры там проще.
### Qbittorent
До того, как я начал реализовывать проект, у меня уже стоял qbittorent установленный через магазин приложений TOS, мне было очень лень его выпиливать, я так и оставил. На просторах сети, умные люди поднимают его в одном `docker-compose` с другими сервисами нашей установки, но мы не ищем простых путей. Я добавлю блок qbittorent для docker-compose, если вы хотите пойти путем попроще.
```yaml
qbittorrent:
  image: linuxserver/qbittorrent
  container_name: qbittorrent
  environment:
    - WEBUI_PORT=8068
    - PUID=1004 #впишите свои значения найденные в блоке "подготовка"
    - PGID=1004
    - TZ=Europe/London
    - WEBUI_PORT=6004
  volumes: #поменийте пути на валидные для вас.
    - ./qbittorrent:/config 
    - ./downloads:/downloads 
  ports:
    - 6881:6881
    - 6881:6881/udp
    - 8068:8068
  restart: always
```
Если ставили qbittorent, через магазин, веб-морда будет на порте 8282. Логин по дефолту будет admin, пароль adminadmin. Рекомендую сменить. В случае с устровкой через docker, смотрите пароль в логах контейнера `docker logs qbittorrent`. С торентом пока все.
### Wireguard
> [!warning] 
>  Данный шаг предпологает, что вы обладаете конфигурационным файлом WG, либо имеете возможность его создать.

Без VPN или proxy, ваши jacket, sonarr, radarr не смогут выполнять свои функции (не во всех странах конечно), но трафик всего NAS заворачивать в тунель решение мягко говоря "из говна". Мы будем туннелировать только вышеперечислленные сервисы, qbittorent не заворчиваем, чтобы не упираться в ширину канала тунеля. Начинаем писать наш docker-compose. В удобной для вас директории создайте файл docker-compose.yaml.
Данный образ не лишен минусов, он чуть более сложный для новичка и в моем случае с ipv6 отказывался работать, НО оно умеет не тунелировать вашу домашнюю подсеть, а это - здорово!
- принудительно отрубаем ipv6, в конфиге WG удалите ::/0 из болка AllowedIPs
- в переменную LOCAL_SUBNETS впишите свою домашгюю подсеть.
- заметьте я запускаю привелигерованный контейнер, с точки безопасноти это - поминки, я делал на скорую руку. Протестирую на выхах, стартанет ли без этой опции.
- знаю, что latest тег использовать не хорошо, но я плохой мальчик.
- т.к. в дальнешем остальные сервисы будут использовать сеть контейнера wireguard порты для приложений пробрасываются в его конфиге.
```yaml
services:
  wireguard:
    image: ghcr.io/captmicr0/docker-wireguard:latest 
    container_name: wireguard
    privileged: true
    cap_add:
      - ALL
    sysctls:
      - net.ipv4.conf.all.src_valid_mark=1
      - net.ipv4.ip_forward=1
      - net.ipv6.conf.all.disable_ipv6=1 
    environment:
      - LOCAL_SUBNETS=192.168.51.0/24
    volumes:
      - /Volume1/docker/containers/wireguard/config:/etc/wireguard #сюда кидаем конфиг, если файлов несколько, берется первый по алфвиту. 
      - /lib/modules:/lib/modules:ro
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "wg", "show"]
      interval: 60s
      timeout: 15s
      retries: 5
      start_period: 60s
    mem_limit: 128m
    mem_reservation: 32m
    cpu_shares: 256
    ports:
      - 6006:9117 #порт для Jackett
      - 6003:7878 #порт для Radarr
      - 6004:8989 #Sonarr
```

Стартуем docker-compose up -d, ждем пока выкачаются образа и контейнеры поднимутся. Проверяем, если в ответе получили ip вашего vpn сервера - успех.
```bash
docker exec -it wireguard wget -qO- icanhazip.com
```

### Jackett

Дописываем в docker-compose.yaml, стартуем docker-compose up -d
```yaml
  jackett:
    image: linuxserver/jackett:latest
    container_name: jackett
    depends_on:
      wireguard:
        condition: service_healthy
    volumes:
      - /Volume1/docker/containers/jackett:/config
      - /Volume1/films:/media #директория куда будут складываться наши фильмы
    environment:
      - PUID=1004 #впишите свои значения найденные в блоке "подготовка"
      - PGID=1004
      - TZ=Europe/Moscow
    network_mode: "service:wireguard"
    restart: unless-stopped
#    restart: always
    mem_limit: 512m
    mem_reservation: 256m
    cpu_shares: 512
```

Дожидаемся пока контейнер поднимется. проверяем что wed-интерфейс доступен http://ip-your-NAS:6006/. Добавьте необходимые индексеры, нажав Add indexer, процесс интуитивно понятный, для нормального поиска контента на русских трекерах рекомендуют проставить галочку Strip Russian Letters. Так же скопируйте api key из правого верхнего угла экрана и сохраните, он нам скоро понадобится. Далее, идем в настройки Radarr\Sonarr. Настройки-индексеры(indexers)

### Radarr

Дописываем в docker-compose.yaml, стартуем docker-compose up -d
```yaml
  radarr:
    image: linuxserver/radarr:latest
    container_name: radarr
    depends_on:
      wireguard:
        condition: service_healthy
    network_mode: "service:wireguard"
    environment:
      - PUID=1004 #впишите свои значения найденные в блоке "подготовка"
      - PGID=1004
      - TZ=Europe/Moscow
    volumes:
      - /Volume1/docker/containers/radarr:/config
      - /Volume1/films:/films #директория куда будут складываться наши фильмы
      - /Volume1/qBittorrent/qBittorrent/downloads:/downloads #путь до директории, куда скачивает файлы торент. Его мы вписываем в Remote Path Mappings
    restart: unless-stopped
    mem_limit: 512m
    mem_reservation: 256m
    cpu_shares: 768

```
Дожидаемся пока контейнер поднимется. проверяем что wed-интерфейс доступен http://ip-your-NAS:6003/. Логинемся, в настройках (Settings) во вкладке Download Client добавить QBitTorrent. Жмём большой плюс, выбираем QBitTorrent и вводим название, хост, порт, логин, пароль, жмем "тест" и сохраняем в случае успеха. Тут же в секиции Remote Path Mappings в самом низу, жмем плюс, откроется форма. В поле host вписывем ip вашего NAS, можно так же использвать loopback 127.0.0.1, в следующих двух полях вписываем пути до папки с загрузками на хостовой машине и в контенере соотвестенно /Volume1/qBittorrent/qBittorrent/downloads и /downloads. Можно ребутнуть контейнер, чтобы пооубавилось предупреждений в меню. Так же в настройках есть русский язык, если кому удобней.
Так же нужно добавить индексер, аналоогичено Settings/Indexers, жмем плюсик, выбираем Torznab и заполняем соглавсно картинке взятой [из статьи](https://connect.smartliving.ru/profile/225/blog/ustanovka-i-nastroyka-sonarr-radarr-qbittorrent-jackett-torproxy-v-docker.html), так же рекомендую указывать категории для верного поиска и приоретет индексера, если вы добавили несколько. Для этого есть доп настройки, нужно нажать на шестеренку снизу. Тут тоже можно провести тест соединения.

Можно сделать чтобы Radarr и Sonarr обновляли библиотеку PLEX.
1. **Откройте Sonarr** → перейдите в **Settings** → **Connect**.
2. Нажмите **+** и выберите **Plex Media Server**.
3. Заполните поля:
    - **Name**: `Plex` (можно любое)
    - **Host**: `192.168.XXX.XXX` (IP вашего NAS, где установлен Plex) 
    - **Port**: `32400` (стандартный порт Plex)
    - **Username** и **Password** (от вашего аккаунта Plex)
4. **Важно:** поставьте галочку **"Update Library"** (Обновить библиотеку) — именно она заставляет Plex сканировать папки после импорта
5. В разделе **"Triggers"** (Триггеры) я рекомендую отметить:
    - **On Import** (При импорте) — обновляет Plex, когда эпизод готов
    - **On Upgrade** (При обновлении) — если скачалась версия лучше
    - **On Rename** (При переименовании) — если вы меняете названия файлов
6. Нажмите **Test** (тест). Если все заполнено верно, вы увидите зеленое уведомление "Success". Затем нажмите **Save**.
Если вы не хотите или не можете указывать пароль Plex в Sonarr, можно использовать [**Plex Token**](https://docs.ultra.cc/unofficial-application-installers/autoscan#main-content). Получите его можно в настройках PLEX, вставьте его в поле **Token** вместо логина/пароля.
